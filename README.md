## Catatan Pembelajaran
**Dependency Inversion Principle (DIP) → High-level module harus depend pada abstraksi, bukan implementasi konkret.**
**Abstraction over implementation → Interface memisahkan apa yang dilakukan dan bagaimana cara melakukannya.**
**Loose coupling → Memudahkan penggantian implementasi tanpa ubah kode high-level.**
**Dependency Injection → Cara menyuntikkan dependency dari luar agar modul lebih fleksibel dan testable.**
**Testable & maintainable → Dengan DIP, modul bisa diuji secara terpisah dan lebih mudah dikembangkan atau diperluas.**
